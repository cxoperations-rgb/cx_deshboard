# cx-대시보드 작업일지

## 2026-05-21 — 초기 구축 및 Vercel 배포 완료

### 작업 목적
- `/최신화` 명령어 실행 시 자동으로 HTML 대시보드와 사본 CSV를 재생성
- 생성된 CSV를 팀 공유 URL에 업로드해두면 재배포 없이 대시보드 데이터 갱신 가능

---

### 구현 내용

#### 1. 대시보드 HTML (`index.html`)
Apple 디자인 시스템 기반 반응형 대시보드. Chart.js 4.x + Papa Parse.

**탭 구성**
- 그래프 탭: 3개 차트
  - ① 배정건 — 완료/추적 진행중 (가로 누적 막대, 월별/8주/14일 토글)
  - ② 성공/종결 비중 추이 (누적 막대, 신청유도/신청후작성/부재종결/신청취소/기타)
  - ③ 내재화 접수 건수 + 기여율 추이 (이중 Y축: 건수 막대 + 기여율 꺾은선)
- KPI 카드 탭: 최근 3개월 / 월별 토글 / 분기별 토글 (직전 기간 대비 증감 표시)

**데이터 로딩 흐름**
1. 페이지 로드 시 `/api/data` 호출
2. 404 응답 → "업로드된 데이터가 없습니다" + 업로드 버튼
3. 200 응답 → CSV 파싱 → 차트/KPI 렌더링
4. 헤더 업로드 버튼 → `/api/upload`로 전송 후 즉시 로컬 파싱하여 화면 갱신

#### 2. Vercel 서버리스 API
| 파일 | 역할 |
|---|---|
| `api/data.js` | Vercel Blob에서 `dashboard-data.csv` 조회 후 반환. 없으면 404 |
| `api/upload.js` | POST로 받은 CSV를 Vercel Blob에 저장 |

- 런타임: Node.js (CommonJS, `module.exports`)
- Blob 인증: `BLOB_READ_WRITE_TOKEN` 환경변수 (Vercel 프로젝트에 설정됨)

#### 3. 사본 CSV (`대시보드_데이터.csv`)
`대시보드_생성.py`가 전체 CSV에서 개인정보를 제거하고 지표 계산에 필요한 컬럼만 추출하여 생성.
29개 컬럼 (배정일, 완료일, _m, _q, _w, 플래그 a~i, 성공, 기타, 내재화총, 율 컬럼 12개).

#### 4. `/최신화` 명령어 5단계
`python 대시보드_생성.py` 실행 → `지표_대시보드.html` (로컬 확인용) + `대시보드_데이터.csv` (업로드용) 생성.

### 디버깅 이력

| 증상 | 원인 | 해결 |
|---|---|---|
| `/api/data` 404 (배포 직후) | vercel.json `builds`+`routes` 설정이 라우팅 충돌 | `functions` 형식으로 변경 |
| 페이지 "데이터 로딩 중..." 고착 | `cfg2`, `cfg3` 함수의 return 객체 닫는 `}`와 `;` 순서 오류 → 스크립트 전체 파싱 실패 | `});}` → `})};` 로 수정 |
| Blob 연결 오류 가능성 | `api/data.js`에 try-catch 없어 에러 시 응답 없이 함수 종료 | try-catch 추가, 에러 시 500 반환 |
